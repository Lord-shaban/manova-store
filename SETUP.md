# 🚀 تجهيز ونشر متجر مانوفا — خطوة بخطوة (مجاني 100%)

المتجر بقى **موقع ثابت بالكامل** (ملفات HTML/CSS/JS بس) بيتكلم مباشرة مع **Firebase**:

- **Firestore** → المنتجات والأقسام والطلبات والإعدادات وصور المنتجات المرفوعة.
- **Firebase Authentication** → تسجيل دخول لوحة التحكم.
- **Cloudflare Pages** → استضافة الموقع نفسه.

كل ده على الخطط المجانية — **من غير بطاقة ائتمان**.

---

## الخطوة 1: إنشاء مشروع Firebase

1. ادخل على [console.firebase.google.com](https://console.firebase.google.com) وسجّل بحساب جوجل.
2. اضغط **Create a project** (أو "إنشاء مشروع") → سمّيه مثلًا `manova-store`.
3. لما يسألك عن Google Analytics → اقفله (مش محتاجينه) → **Create project**.

## الخطوة 2: إضافة تطبيق ويب وأخذ الإعدادات

1. من صفحة المشروع اضغط على أيقونة الويب **`</>`** (Add app → Web).
2. سمّيه `manova` → **Register app** (من غير Firebase Hosting).
3. هيظهرلك كود فيه `firebaseConfig` — انسخ القيم اللي جواه.
4. افتح الملف **`public/js/firebase-config.js`** في المشروع وحط القيم مكان العلامات:

```js
window.FIREBASE_CONFIG = {
  apiKey: 'AIza....',
  authDomain: 'manova-store.firebaseapp.com',
  projectId: 'manova-store',
  storageBucket: 'manova-store.appspot.com',
  messagingSenderId: '1234567890',
  appId: '1:1234567890:web:abcdef',
};
```

## الخطوة 3: تفعيل تسجيل الدخول (Authentication)

1. من القائمة الجانبية: **Build → Authentication** → اضغط **Get started**.
2. من تبويب **Sign-in method** → فعّل **Email/Password** → **Save**.
3. من تبويب **Users** → اضغط **Add user** → اكتب إيميلك وكلمة مرور قوية → **Add user**.
4. **مهم:** بعد إضافة المستخدم هتلاقي عمود **User UID** — انسخ الـ UID ده (سلسلة حروف وأرقام طويلة)، هنحتاجه في الخطوة الجاية.

## الخطوة 4: إنشاء قاعدة البيانات (Firestore)

1. من القائمة الجانبية: **Build → Firestore Database** → **Create database**.
2. اختر الموقع الجغرافي (مثلًا `eur3` أو أي موقع قريب) → اختر **Start in production mode** → **Create**.
3. من تبويب **Rules** → امسح الموجود والصق محتوى ملف **`firestore.rules`** من المشروع كله → **Publish**.

## الخطوة 5: إضافة حسابك كأدمن

القواعد بتسمح بالإدارة بس للمستخدمين الموجودين في مجموعة `admins`:

1. من تبويب **Data** في Firestore → اضغط **Start collection**.
2. Collection ID: اكتب `admins` بالظبط → **Next**.
3. في خانة **Document ID**: الصق الـ **UID** اللي نسخته في الخطوة 3.
4. أضف حقل واحد: Field = `role` / Type = `string` / Value = `owner` → **Save**.

## الخطوة 6: رفع الموقع على Cloudflare Pages

1. اعمل حساب مجاني على [dash.cloudflare.com](https://dash.cloudflare.com).
2. من القائمة: **Workers & Pages → Create → Pages**.
3. عندك طريقتين:
   - **ربط GitHub (الأفضل):** اختر **Connect to Git** → اختر ريبو المتجر → في الإعدادات:
     - Build command: *(سيبها فاضية)*
     - Build output directory: **`public`**
   - **رفع مباشر:** اختر **Upload assets** → اسحب محتويات مجلد **`public`** كلها.
4. اضغط **Deploy** — خلال دقيقة هيبقى عندك رابط زي `https://manova.pages.dev`.

> **ملحوظة:** لو ربطت GitHub، أي تعديل هترفعه للريبو هيتنشر تلقائيًا. وتقدر بعدين تربط دومين خاص بيك مجانًا من تبويب **Custom domains**.

## الخطوة 7: تجهيز بيانات المتجر (آخر خطوة!)

1. افتح رابط موقعك + `/admin/login` وسجّل دخول بالإيميل والباسورد.
2. افتح **`/admin/setup`** — الصفحة هتفحص كل حاجة وتقولك الوضع.
3. اضغط **"تجهيز المتجر بالبيانات الأولية"** — هيستورد المنتجات الـ 14 والأقسام والإعدادات (والطلبات التجريبية لو سايب العلامة).

**مبروك! 🎉 المتجر شغال بالكامل.**

## 📌 ملاحظات مهمة بعد التشغيل

### تعديل بيانات التواصل
من لوحة التحكم → **الإعدادات**: غيّر رقم الموبايل والواتساب والعنوان ومناطق الشحن وأسعارها.

### مسح الطلبات التجريبية
لما تجهز تستقبل طلبات حقيقية: لوحة التحكم → النظرة العامة أو الإعدادات → **مسح الطلبات التجريبية**.

### صور المنتجات
- الصور اللي بترفعها من لوحة التحكم بتتضغط تلقائيًا في المتصفح (WebP حتى 900px) وبتتخزن جوه Firestore نفسها — عشان كده مفيش حاجة اسمها Firebase Storage هنا (لأنه بقى محتاج بطاقة ائتمان للمشاريع الجديدة).
- صور واضحة بخلفية بسيطة بتضغط أحسن. لو ظهرلك "الصورة كبيرة جدًا" جرّب تقصها قبل الرفع.

### المخزون
المخزون بيتخصم لما **تأكّد** الطلب من لوحة التحكم (مش لحظة ما العميل يطلب) — وبيرجع تلقائيًا لو لغيت طلب مؤكد. ده لأن الموقع من غير سيرفر، فالعميل مش مسموحله يعدّل المخزون بنفسه.

### مراجعة الطلبات
زي أي متجر دفع عند الاستلام: راجع تفاصيل الطلب مع العميل في مكالمة التأكيد (الأصناف والإجمالي) قبل ما تحوله "مؤكد".

### الحدود المجانية (أكتر من كافية لمتجر صغير)
| الخدمة | الحد المجاني اليومي |
|---|---|
| قراءات Firestore | 50,000 قراءة/يوم |
| كتابات Firestore | 20,000 كتابة/يوم |
| تخزين Firestore | 1 GB إجمالي |
| Cloudflare Pages | زيارات غير محدودة |

### نسيت كلمة المرور؟
من صفحة تسجيل الدخول → **"نسيت كلمة المرور؟"** — هيبعتلك رابط على إيميلك. (لازم تكون كتبت الإيميل الأول في الخانة.)

### تشغيل معاينة محلية على جهازك
```bash
npm run dev
# أو دبل كليك على: تشغيل-المتجر.bat
```
وافتح `http://localhost:3000` — المعاينة بتتصل بنفس قاعدة بيانات Firebase الحقيقية.

### بديل: النشر على Firebase Hosting بدل Cloudflare
لو حابب تخلي كل حاجة عند جوجل:
```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # اختار المشروع بتاعك، و public كمجلد، وقول No للـ single-page app
firebase deploy
```
ملف `firebase.json` جاهز بالفعل بالإعدادات الصحيحة.
