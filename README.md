متجر إلكتروني متكامل لبراند **مانوفا** — تيشرتات بيسيك ومطبوعة بخامات قطن مختارة، الدفع عند الاستلام، والتوصيل لكل المحافظات. مبني بأدوات بسيطة وسريعة بدون أي أطر أمامية ثقيلة، ومعاه لوحة تحكم كاملة لإدارة المنتجات والأقسام والطلبات والإعدادات.

## ✨ المميزات

### 🛍️ المتجر (للعملاء)
- صفحة رئيسية بهوية البراند (ذهبي/أسود) مع أقسام ومنتجات مميزة.
- تصفّح المنتجات بالأقسام (بيسيك / مطبوع / وأي قسم تضيفه) مع **بحث وفرز**.
- صفحة منتج باختيار المقاس واللون والكمية ومعرض صور.
- سلة تسوق (تُحفظ في المتصفح) + إتمام الطلب بالدفع عند الاستلام أو المحفظة الإلكترونية.
- **تتبّع الطلب** برقم الطلب ورقم الموبايل.
- تصميم متجاوب بالكامل (موبايل/تابلت/كمبيوتر) وواجهة عربية RTL.

### 🎛️ لوحة التحكم (للإدارة)
- **نظرة عامة**: إحصائيات المبيعات، رسوم بيانية يومية، وأكثر المنتجات مبيعًا.
- **الطلبات**: عرض وبحث وتغيير حالة الطلب (جديد → مؤكد → شحن → توصيل)، مع إرجاع المخزون تلقائيًا عند الإلغاء.
- **المنتجات**: إضافة/تعديل/حذف، رفع صور، مقاسات وألوان، مخزون، وتمييز المنتج.
- **الأقسام**: نظام أقسام ديناميكي — أضِف «بناطيل» أو «كابات» أو «أحزمة» أو «جزم»… وتظهر تلقائيًا في المتجر.
- **الإعدادات**: محتوى الصفحة الرئيسية، بيانات التواصل، مناطق الشحن وأسعارها، وتغيير كلمة المرور.

## 🛠️ التقنيات

- **Node.js (≥18) + Express 5** — السيرفر و REST API.
- **تخزين JSON بسيط** — بدون قاعدة بيانات خارجية؛ كتابة ذرّية آمنة (`lib/db.js`).
- **Multer** — رفع صور المنتجات.
- **HTML/CSS/JS خام** — بدون أي framework أمامي، خفيف وسريع، RTL بالكامل.
- **بدون أي اعتماد على خدمات خارجية** — يشتغل بالكامل من ملفاته.

## 🚀 التشغيل محليًا

**المتطلبات:** [Node.js](https://nodejs.org) نسخة 18 أو أحدث.

```bash
# 1) تثبيت الحزم
npm install

# 2) التشغيل
npm start
```

ثم افتح المتصفح على:

| الرابط | الوصف |
|---|---|
| http://localhost:3000 | المتجر (اللي يشوفه العميل) |
| http://localhost:3000/admin | لوحة التحكم |

> على ويندوز تقدر كمان تعمل دبل كليك على ملف `تشغيل-المتجر.bat`.

**بيانات الدخول الافتراضية:** المستخدم `admin` / كلمة المرور `manova123`
> ⚠️ غيّرها فورًا من: لوحة التحكم ← الإعدادات ← تغيير كلمة المرور. (وفي الاستضافة اضبط `ADMIN_PASSWORD` — انظر بالأسفل.)

عند أول تشغيل يتم إنشاء `data/db.json` تلقائيًا من البيانات المبدئية في `lib/seed.js` (14 منتج + طلبات تجريبية لعرض الداشبورد).

## ⚙️ متغيرات البيئة

كل المتغيرات اختيارية محليًا؛ لكنها مهمة على الاستضافة. انسخ `.env.example` إلى `.env`:

| المتغير | الوصف | الافتراضي |
|---|---|---|
| `PORT` | منفذ التشغيل | `3000` |
| `DATA_DIR` | مسار حفظ قاعدة البيانات (اضبطه على القرص الدائم في الاستضافة) | `./data` |
| `UPLOADS_DIR` | مسار حفظ الصور المرفوعة | `./public/uploads` |
| `ADMIN_USERNAME` | اسم مستخدم المدير الأول | `admin` |
| `ADMIN_PASSWORD` | كلمة مرور المدير الأولى | `manova123` |

## ☁️ النشر على Render

المشروع بيحفظ الطلبات والصور كملفات، فلازم استضافة **بقرص دائم** (Persistent Disk) عشان الطلبات متتمسحش مع كل تحديث. ملف [`render.yaml`](render.yaml) جاهز لكل ده.

1. ارفع الريبو على GitHub (تم ✅).
2. من [render.com](https://render.com) اعمل حساب واربطه بجيت هب.
3. اضغط **New + → Blueprint** واختر هذا الريبو — Render هيقرأ `render.yaml` تلقائيًا (خدمة ويب + قرص دائم 1GB على مسار `/var/data`).
4. من تبويب **Environment** اضبط `ADMIN_USERNAME` و`ADMIN_PASSWORD` بقيم قوية.
5. **Deploy** — وخلال دقائق يكون المتجر شغّال على رابط `https://manova-store.onrender.com` (أو رابطك).

> ملاحظة: القرص الدائم متاح على الخطة المدفوعة (Starter). الخطة المجانية تشتغل لكن **بتفقد الطلبات مع كل نشر** — مناسبة للتجربة فقط.

## 🗂️ هيكل المشروع

```
manova/
├── server.js            # السيرفر و REST API
├── lib/
│   ├── db.js            # طبقة تخزين JSON (كتابة ذرّية)
│   └── seed.js          # البيانات المبدئية (منتجات + أقسام + طلبات تجريبية)
├── public/              # واجهة المتجر (HTML/CSS/JS)
│   ├── index.html · shop.html · product.html · cart.html
│   ├── checkout.html · track.html · success.html · 404.html
│   ├── css/store.css · js/store.js
│   └── images/products/ # صور المنتجات
├── admin-ui/            # واجهة لوحة التحكم
│   ├── index.html · orders.html · products.html
│   ├── categories.html · settings.html · login.html
│   └── assets/          # admin.css · admin.js · charts.js
├── data/db.json         # قاعدة البيانات (تُنشأ تلقائيًا — غير مرفوعة)
├── render.yaml          # إعداد النشر على Render
└── .env.example         # نموذج متغيرات البيئة
```

## 🔐 الأمان

- بدّل كلمة مرور المدير فور أول دخول، واضبط `ADMIN_PASSWORD` في الاستضافة.
- `data/db.json` (فيه بيانات الطلبات) و`.env` **غير مرفوعين** على جيت هب (`.gitignore`).
- الأسعار وحسابات الطلب تُحسب في السيرفر دائمًا، مش من طرف العميل.
